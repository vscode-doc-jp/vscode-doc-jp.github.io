何百ものプログラミング言語をサポート

Visual Studio Code では主要なプログラミング言語をほとんどサポートしています。例えば JavaScript, TypeScript, CSS , HTML また他の言語拡張を VS Code Marketplace で探して取り寄せることができます。

VS Code の言語機能

言語と拡張機能によって、サポートの豊かさが異なることに注意してください。

  • Syntax highlighting と bracket matching
  • 高性能な補完 (IntelliSense)
  • Linting と corrections
  • Code navigation (定義へ移動 , 全ての参照検索)
  • Debugging
  • Refactoring

言語拡張機能

コミュニティは VS Code における現代プログラミング言語をサポート提供しています。興味のある言語の拡張機能 があるか見るには、単純に Extensions view (⇧⌘X (Windows, Linux Ctrl+Shift+X)) を開いて、検索ボックスに言語の名前でフィルターをかけるだけです。

また、 VS Code Marketplace で サポートされている言語 を探すこともできます。

ファイルの言語を変更

VS Code は、ファイル拡張子に基づいて言語モードを選択します。言語モードを変更したくなったら、ステータスバー右にある言語モードの選択をクリックします。これで、言語選択モードのパレットが表示されるはずです。

言語 ID

VS Code は現在の言語モードを確定するために、特定の識別子と言語モードを関連付けます。

言語識別子は、言語の名前の小文字であることがそれなりにあります。しかし例えば、Markdownがmarkdownでないことに注意してください。また、未知の言語ファイルは plaintext(プレーンテキスト) として扱われます。

既知の識別子リストは [言語識別子リファレンス](/docs/languages/identifiers.md] で確認できます。

言語モードの変更 (⌘K M (Windows, Linux Ctrl+K M)) で既にインストールされた言語と識別子を確認できます。

よくある質問

Q: 私が言語サービスを提供することはできますか?

A: Yes you can! もちろんです Visual Studio Code 拡張 ドキュメントの example language server を確認してください。

Q: 完全な言語サービスを作成したくない場合 既存の TaxtMate を再利用できますか?

A: はい TextMate colorizers を通して好きな言語サポートを追加することもできます　。Colorizers で VS Code に TextMate .tmLanguage 構文ファイルを統合する方法を確認してください。

Q: 言語にファイル拡張子を追加することができますか?

A: はい files.associations setting でグローバルでもワークスペース内でも拡張子を追加することで可能です。
