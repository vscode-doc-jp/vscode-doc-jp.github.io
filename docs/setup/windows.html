Windows で VS Code を起動する

Last updated: 2017-02-20

インストール

  1. Visual Studio Code installer を入手する
  2. ダウンロードを完了したら、インストーラー(VSCodeSetup-stable.exe)を実行します これには1分ほどかかります
  3. 既定では、64bitの場合 C:\Program Files (x86)\Microsoft VS Code にインストールします

Zip archive はこちら

Note: 起動には .NET Framework 4.5.2 が必要です。Win7 を利用している場合には .NET Framework 4.5.2 がインストールされていることを確認してください。

Tip: オプションでcodeを環境変数に追加することが可能です。追加すればそのフォルダから簡単にVS Code を呼び出せます。ただし、コンソールで環境変数の変更を有効にするには、インストール後にコンソールを再起動する必要があります。

アップデート

VS Code は毎月新しいリリースを提供しています。もし新しい更新が利用可能になれば VS Code は自動更新します。新しい更新の準備が整えば、 VS Code でプロントが表示されますので、それに沿って更新してください(最新のものを入手するのに何の準備をする必要はありません)。 VS Code の更新を手動で行う場合は、How do I opt out of auto-updates を参照します。

インストーラーのトラブル

インストーラーの代わりに zip file を試してください。この場合には、VS Code を Program Files に解凍します。(解凍場所は自由)

Note: Zip ファイルによるインストールの場合には、新しいリリース ごとに手動で更新する必要があります。

アイコンが不足しています

Win7, 8 のマシンに VS Code をインストールしました。なぜ一部のアイコンがワークベンチやエディタに表示されませんか?

VS Code のアイコンはSVGによって実現しています。ですから、.SVG は image/svg+xml と関連付けられている必要があります。私たちはこれを修正するオプションを検討していますが、現在は次の回避策を試します。

コマンドプロントによる修正:

  1. 管理者コマンドプロントを開く
  2. REG ADD HKCR\.svg /f /v "Content Type" /t REG_SZ /d image/svg+xml を入力

レジストリエディタ(regedit)による修正:

  1. regedit を実行
  2. HKEY_CLASSES_ROOT を開く
  3. .svg を探し出す
  4. Content Type の値を image/svg+xml に設定
  5. regedit を終了
