Visual Studio Code にスニペットを追加

Last updated: 2017-03-06

コードスニペットとは、ループや条件文など繰り返し使うコードパターンを簡単に生成するテンプレートのことです。

スニペットは IntelliSense(⌃Space (Windows, Linux Ctrl+Space)) に他の提案や専用のスニペット選択(Insert Snippet)と組み合わせて表示します。タブ補完サポートもしています。"editor.tabCompletion": true を有効にして snippet prefix を入力したら Tab を押してスニペットを挿入可能です。

スニペット構文をサポートしていない ‘regular expression replacements’, ‘interpolated shell code’, ‘transformations’ を除いて、TextMate snippet syntax に従います。

Marketplace からスニペットを追加

VS Code Marketplace には多くの拡張機能を用意しています。使いたいものをインストールして再起動すれば、新しいスニペットが利用可能になります。(拡張機能インストールの詳細はここを参照してください)

独自のスニペットを作成

特定の言語に独自のスニペットを定義することができます。ファイル > 基本設定 > ユーザースニペット(Code > 基本設定 > ユーザースニペット)を開いて、スニペットを定義可能です。

スニペットは JSON 形式で定義し、ユーザーごとの (languageId).json に保存します。例えば Markdown スニペットは、 markdown.json に登録します。

スニペット シンタックス

body は特殊な構文を使ってカーソルと挿入テキストを制御できます。サポートする機能と構文を次に示します。

タブストップ

タブストップを使用すると、カーソルをスニペットの指定場所に移動させることができます。カーソル位置を $1, $2 のように指定します。番号順にカーソルを移動しますが、 $0 は最終的なカーソル位置を示します。同じ番号のタブストップは、入力文字をリンクし同期します。

プレースホルダー

プレースホルダーとは ${1:foo} のような値を持つタブストップです。 まずプレースホルダーテキストを入力して、容易に内容を変更できるように選択状態になります。プレースホルダーは ${1:another ${2:placeholder}} のように入れ子にすることができます。

変数

$name や ${name:default} を使うことで、変数の値を挿入することができます。変数を設定していないときは、そのdefaultまたは空文字列を挿入します。変数が不明な場合(名前が定義されていない場合)、変数名を挿入してプレースホルダーに変換します。次の変数を使用できます。

TextMate Snippets を使用する

また TextMate スニペット(.tmSnippets) を拡張機能ジェネレータ yo code を使って追加できます。パッケージ化には、.tmSnippets (複数可) を指定してパッケージ化する New Code Snippets オプションを使用します。またジェネレータは、Sublime スニペット(.sublime-snippets)もサポートします。

最終的な出力は、スニペットを VS Code に統合するための拡張機能マニフェスト package.json と VS Code スニペット形式に変換した snippets.json になります。

出力ファイルを .vscode/extensions フォルダにコピーして VS Code を再起動します。

Marketplace でスニペットを公開

スニペットを作成してテストしたら、コミュニティと共有することができます。

スニペットをスニペット拡張機能にする必要があります。もちろん yo code 拡張ジェネレータを使用していれば、拡張機能はもう公開できているも同じです。(翻訳メモ:この2文がさす意味がわかからない

スニペットを共有する場合は、VS Code に結合する拡張マニフェストと共に snippet json をパッケージする必要があります。

プラットフォーム別ユーザー スニペット ファイルは次の場所にあります:

  • Windows %APPDATA%\Code\User\snippets\(language).json
  • Mac $HOME/Library/Application Support/Code/User/snippets/(language).json
  • Linux $HOME/.config/Code/User/snippets/(language).json

ここで (language).json はスニペットの対象言語(e.g markdown.json -> markdown snippet) にしか対応しません。拡張機能用の新しいフォルダを作成し、スニペットファイルを snippets サブディレクトリにコピーします。

拡張機能マニフェスト package.json を extension フォルダにコピーします。拡張機能マニフェストは Extension Manifest で定義された構文にしたがって snippets contribution を提供します。

Markdown スニペットのマニフェスト例です:

スニペットは language 識別子に関連付ける必要があります。これは VS Code または、拡張機能の言語提供による言語サポートのうちである必要があります。必ず language 識別子が正しいことを確認してください。

そうしたら vsce publishing tool を使用して、VS Code Extension Marketplace に公開します。

Tip: ユーザがあなたのスニペットを見つけやすくするために、拡張機能 package.json の説明に “snippet” という単語を入れて Category を Snippets に設定します。

VS Code Marketplace で 拡張機能を素晴らしいものにする方法についての提案を Marketplace Presentation Tips を参照してください。

よくある質問

Q: 作成したスニペットを VS Code エディタに表示しませんか?

A: スニペットの language 識別子を正しく指定してください(e.g. Markdown, .md なら markdown、 Plain Text, .txt なら plaintext)。また、 snipets json のへの相対パスが正しいことも確認してください。
