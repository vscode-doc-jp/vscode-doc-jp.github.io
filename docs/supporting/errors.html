よくあるエラーケース

Last updated: 2017-02-20

Visual Studio Code で起こるエラーのいくつかは、回避したり解決したりすることが可能です。このトピックでは、よくあるエラーとその解決方法を説明します。(編集メモ: 現時点でこのエラーケースが役に立つとは言えない。一種の慣用的表現)

もし、これらの手段で問題が解決しないのなら、バグを目覚めさせてしまった可能性があります。報告された問題 を確認して、他の人と同じ問題を抱えていないかどうかを確認してください。

20002

Error: Node.js を起動するための ‘/usr/bin/gnome-terminal’ が見つかりません。

Linux の VS Code で Node.js デバッガを使って Node.js を起動するには gnome-terminal が必要です。

gnome-terminal がインストールされていない場合、 VS Code デバッガはデバッグのためにプログラムを起動できません。

この問題を解決するには、次の2つの方法があります。

  • sudo apt-get install gnome-terminal (もしくは Linux distribution と等価) を実行して gnome-terminal をインストールします。
  • Node.jsに --debug か debug-brk オプションを設定して、デバッグモードでプログラムを手動起動する。次に、 VS Code デバッガを port 5858 にアタッチする。

20003

Error: program が絶対パスではありません。’${workspaceRoot}/‘ を先頭に追加して絶対パスにすることを検討してください。

VS code (release 0.10.11)より前は、起動設定で相対パスを使用できました。そのために VS Code は暗黙的に絶対パスに変換してきました。

しかしこれには次2つの問題があります:

  • VS Code は program, cwd, outFiles のようなよく知られている属性のパスのみ修正します。故に引数, 環境変数に設定される相対パスは固定されず、この動作は不透明でした。
  • VS code は lunch.json のパスは修正しました。しかし tasks.json を修正せず、この矛盾はわかりづらいものでした。

結果的に、0.10.11 から VS Code は起動設定パスを変更しなくなりました。このことで、起動設定で相対パスを利用している場合は、相対パスの先頭に${workspaceRoot}/ を付けて絶対パスに修正する必要があります。
